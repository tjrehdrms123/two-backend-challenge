# 2024년 (주) 두번째 백엔드 개발자 채용 과제

- [과제 데이터 샘플 엑셀](./과제_데이터.xlsx)

## 사용 가능 기술

- Node.js: Express 또는 Nest.js
- MySQL 또는 다른 관계형 Database
- 데이터베이스 선택에 따른 ORM
- .xlsx 또는 .csv 파일 다루기 위한 라이브러리

## 준비

1. 구글 시트(링크)로 제공된 가상 데이터를 .xlsx 또는 .csv 파일로 다운로드
2. 엑셀 파일 내의 데이터를 관리하기 위한 데이터베이스 생성 (DDL 제출 필수)

## API 목록

### 1. 고객정보 및 주문내역정보 업로드 API

- 엑셀 파일을 읽어서 Database에 데이터를 등록
- API 요청에 xlsx(또는 csv) 파일을 담아서 DB에 데이터 입력

### 2. 월별 매출 통계 API

- 월별 매출 합계 데이터 리턴
- 주문타입이 "order"인 경우 가산, "refund"인 경우 감산

```
(데이터 예시)

2023년 1월 주문액 : 1,000,000원
2023년 1월 반품액 : 200,000원
2023년 1월 매출 : 800,000원
…
2023년 2월 주문액 : 1,500,000원
2023년 2월 반품액 : 300,000원
2023년 2월 매출 : 1,200,000원
…

```

### 3. 주문 목록 조회 API

- 주문 목록을 조회하는 페이지 가정
- query parameter로 주문 데이터를 조건에 맞게 리턴

#### 조회 조건

- startDate (string): 기간 조회 시작일 (예시: "2024-01-01")
- endDate (string): 기간 조회 종료일 (예시: "2024-01-31")
- orderType (number): 주문 또는 반품 조회 (0: 주문, 1: 반품, 미입력시 주문 & 반품 모두)
- customerId (number): 특정 고객의 주문만 조회 시, 고객 id
- pageSize (number): 페이지네이션 - 한 페이지 조회 건 수 (기본값: 50)
- pageNo (number): 페이지네이션 - 조회할 페이지 번호 (기본값: 1)

#### 1. 리턴 데이터

- 주문일자
- 주문고객명
- 주문고객 등급
- 주문타입 (order / refund)
- 주문금액

#### 2. 필터 적용

- 주문기간(시작일, 종료일) 내의 주문만 조회
- 주문 또는 반품만을 조회 또는 둘 다 조회
- 특정 고객의 주문(또는 반품, 또는 둘 다)만을 조회
  - 필터 a, b, c 는 동시에 적용될 수 있어야 합니다.

#### 3. 페이지네이션 적용

- query parameter로 페이지네이션 값 입력하지 않는 경우, 기본값으로 한 페이지 건 수 50건, 페이지 번호 1페이지 적용
